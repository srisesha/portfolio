CAPGEMINI

Client: HSBC-IHub
Location: HSBC, Hyderabad
Duration: 2.0 Years


This project deals with multiple source systems from different regions. Extract & Load jobs are generic jobs which are common for all the applications. Transformations will be implemented as per the mapping sheet using GCP BigQuery code. Once the code is developed & tested, code will be deployed into Production using Jenkins, GITHUB & G3 deployment tools. Defects & progress will be tracked using the Jira tool.

Roles & Responsibilities:


● Understanding the requirements & mapping sheet. Having discussions with Business to understand the requirements.

● Involved in the modeling and design of data ingestion into GCP Data Warehouse.

● Writing the BigQuery SQL queries to extract the data from the database.

● Have dealt with multiple stages like Transformer, Sequential file, Lookup, Sort, Join, Copy, TeraData connector, Dataset, Change Data capture, surrogate key generator stages .

● Have created the Datastage sequence jobs to control the branch flow of parallel jobs and Unix scripts.

● Have created the Control M Jobs to schedule the DS Jobs as per the run frequency.

● Have prepared the unit test cases, performed testing and captured the test results.

● Have attended the daily scrum calls & weekly scrum of scrum calls to update the progress of stories.

● JIRA has been updated periodically on the progress of each story. Stories, Tasks, Test results and Defects have been logged in JIRA.

● Code has been deployed using deployment tools Jenkins, GITHUB, Control M quality gate & RTC G3.

Environment:


Google Cloud BigQuery, IBM Infosphere DataStage11.5, Unix, Control M, Win SCP, Microsoft Office, Notepad++, Jira, Code Deployment tools like Jenkins, GITHUB, Control M quality gate & RTC G3.
